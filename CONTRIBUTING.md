# Contributing to Spatial Mesh Engine

Thank you for your interest in contributing! We welcome contributions of all kinds.

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test your changes: `./setup.sh && python test_engine.py`
5. Commit with clear messages
6. Push to your fork and open a pull request

## Code Style

- Use C++20 standards
- Follow existing code patterns in `src/core/`
- Use meaningful variable names
- Add comments for complex algorithms

## Areas for Contribution

- **Performance optimization**: Profile and improve solver speed
- **Algorithm improvements**: Better FEM implementations or mesh handling
- **Testing**: Expand test coverage and add benchmarks
- **Documentation**: Improve comments and add examples
- **New features**: Refer to [Roadmap](README.md#-roadmap) for planned work

## Bug Reports

When reporting bugs, please include:
- Minimal reproduction case
- Expected vs. actual behavior
- Your environment (OS, compiler, versions)

## Questions?

Open an issue or start a discussion on GitHub.

Thanks for contributing! 🚀
